ToDo-List for Hackers: (Nearly) done with pure bash-commands

How it was done:

Please note, that this is just a rough (1st) version. I am still relearning my Linux skills and I needed soemthing to play with.

Being fed up with overly web2.0-ish ToDo applications, I thought: "Why not do my own one?" Could not get any worse, but maybe slightly more nerdy.

ToDo, as I call it, was written purely with tools from a "normal" Linux. In this case, I used "Cent OS", as provided by my great hoster "Uberspace".

For the html, I used just "vi". I really dig st old-school tool and it provides me with the raw power, that only a storm-tested tool could bring to the table.

For adding TODOs i just type: 'todo [parameter #1] "parameter #2"', where parameter #1 is being transfered as an id to the li-element and parameter #2 is the TODO, that is shown as the list-element, that is seen on the page.

For deleting TODOs I type: 'tododel [parameter #1]'

Using my .bashrc-File, I let it translate these commands as follows:

  • todo() {
      echo "  <li id="$1">"$2"</li>" >> /var/www/virtual/sdoering/sandbox.moorbekstrasse.de/index.html && sleep 0.5 && lynx sandbox.moorbekstrasse.de
    }
  • tododel() {
      sed -i "/$1/d" /var/www/virtual/sdoering/sandbox.moorbekstrasse.de/index.html && sleep 0.5 && lynx sandbox.moorbekstrasse.de
    }

I am using the id of the li-tag to identify the line, I want to delete with tododel. Practically I am using it as a parameter for tododel.

I know, that I am very much reinventing the wheel here. But it was just done, to learn more and to have a little bit of fun.
